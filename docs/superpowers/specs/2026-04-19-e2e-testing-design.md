# E2E 测试用例

## 测试环境

- 启动: `npm run tauri dev`
- data-testid 属性已添加到关键 UI 元素

---

## TC-01 待办 CRUD

### TC-01.01 创建待办
- 输入框 [data-testid=todo-input] 输入"测试待办A" → Enter
- 断言: 列表出现含"测试待办A"的项 [data-testid=todo-list]
- 断言: Toast 显示「待办已创建」

### TC-01.02 编辑待办
- 双击待办标题 → 修改为"测试待办A-已编辑" → 点击外部
- 断言: 标题已更新，Toast「待办已更新」

### TC-01.03 优先级和截止日期
- 双击进入编辑 → 点击「高」优先级 → 设置截止日期 → 保存
- 断言: 显示红色「高」标签 + 日期文本

### TC-01.04 勾选完成
- 点击待办勾选框 [data-testid=btn-complete-todo]
- 断言: 标题出现 line-through，显示「完成于」

### TC-01.05 取消完成
- 再次点击勾选框
- 断言: line-through 消失

### TC-01.06 创建子任务
- 悬停待办 → 点击 + → 输入"子任务1" → Enter
- 断言: 子任务缩进显示在父任务下方

### TC-01.07 删除待办（确认弹窗）
- 点击 ⋯ 菜单 → 删除
- 断言: AlertDialog 出现，含「删除待办」「不可撤销」文本
- 点击「取消」→ 断言: 待办仍在
- 再次删除 → 点击「删除」
- 断言: 待办消失（淡出动画），Toast「待办已删除」
